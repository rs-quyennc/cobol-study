## DATABASE RELATED OBJECT

### Data Description Specifications (DDS)

DDS là ngôn ngữ định nghĩa cấu trúc cho:

- File vật lý (Physical File – PF)
- File logic (Logical File – LF)
- File hiển thị (Display File – DSPF – màn hình 5250)
- Printer File (PRTF)

DDS - Display File

```
     A          R CUSREC
     A            CUSNO        5Y 0B  5 10
     A            CUSNAME     30A    5 20
     A                                  DSPATR(HI)
```

DDS định nghĩa field, COBOL READ / WRITE / DISPLAY / ACCEPT các field đã được DDS mô tả.

```
SELECT CUSSCR ASSIGN TO WORKSTN.
FD  CUSSCR.
01  SCREEN-REC.
    05 CUSNO     PIC 9(5).
    05 CUSNAME   PIC X(30).
```

### Data Definition Language (DDL)

DDL là ngôn ngữ định nghĩa dữ liệu theo chuẩn SQL, dùng để tạo:

- Table
- View
- Index
- Constraint

```
CREATE TABLE CUSTOMER (
    CUSNO     DECIMAL(5,0),
    CUSNAME   VARCHAR(30),
    BIRTHDAY  DATE
);
```

DDL định nghĩa bảng dữ liệu, COBOL truy cập bằng Embedded SQL hoặc thông qua DB2 for i.

```
EXEC SQL
   SELECT CUSNAME
   INTO :WS-CUSNAME
   FROM CUSTOMER
   WHERE CUSNO = :WS-CUSNO
END-EXEC.
```

### Physical File (PF) và Logical File (LF)

Tham khảo: [Physical Files and Logical Files concept in AS400](https://www.youtube.com/watch?v=dgHYXxq6eRE)

Physical File (PF) và Logical File (LF) là nền tảng của hệ thống quản trị cơ sở dữ liệu DB2. Cách dễ nhất để hiểu là xem PF tương đương với Table, còn LF tương đương với View hoặc Index trong SQL.

| **Mục so sánh**                   | **Physical File (PF)**                                                                                       | **Logical File (LF)**                                                                                                                                     |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Lưu trữ dữ liệu                   | Chiếm dụng một phần bộ nhớ (ổ cứng). PF chứa dữ liệu gốc thực tế.                                            | Không chiếm không gian bộ nhớ lưu trữ dữ liệu. Nó tự nạp vào thời điểm thực thi (run time) dựa theo đường dẫn truy cập đã được định nghĩa.                |
| Định dạng bản ghi (Record format) | Một PF chỉ chứa duy nhất một định dạng bản ghi (record format)                                               | Một LF có thể chứa lên đến 32 định dạng bản ghi khác nhau.                                                                                                |
| Thứ tự dữ liệu                    | Dữ liệu trong PF thường được lưu theo thứ tự xuất hiện (Arrival Sequence) trừ khi định nghĩa một Key cho nó. | LF thường được dùng để định nghĩa một trình tự sắp xếp (Keyed Sequence). Ví dụ: PF lưu theo ID nhân viên, nhưng LF có thể cho bạn xem theo Tên nhân viên. |
| Sự phụ thuộc                      | PF có thể tồn tại mà không cần bất kỳ LF nào.                                                                | LF không thể tồn tại nếu không có PF tương ứng (vì nó phải tham chiếu đến dữ liệu gốc).                                                                   |
| Xóa đối tượng                     | Nếu PF đang có một LF trỏ vào, không thể xóa PF cho đến khi xóa hết các LF liên quan.                        | LF có thể được xóa bất cứ lúc nào mà không ảnh hưởng hay làm mất dữ liệu của PF.                                                                          |
| Lệnh tạo                          | Sử dụng lệnh CRTPF (Create Physical File) để tạo đối tượng này.                                              | Sử dụng lệnh CRTLF (Create Logical File) để tạo đối tượng này.                                                                                            |

### Data File Utility (DFU)

Tham khảo: [Data File Utlity - DFU](https://www.youtube.com/watch?v=vb7B5VogNhc)

DFU là một công cụ giúp nhập, sửa, xóa và xem dữ liệu trong một Physical File một cách nhanh chóng mà không cần phải viết một chương trình COBOL hoàn chỉnh.

DFU cho phép sửa trực tiếp vào Physical rất tiện lợi nhưng nó cũng nguy hiểm nếu sửa nhầm. Do đó, quyền sử dụng lệnh STRDFU (Mở menu chính của công cụ DFU) thường bị hạn chế.

### COPYBOOK

Copybook là một tệp văn bản chứa các đoạn mã nguồn chung (thường là các khai báo field hoặc cấu trúc dữ liệu) được sử dụng để chèn vào nhiều chương trình khác nhau, giúp giữ tính nhất quán giữa các chương trình xử lý giao dịch.

Vd: tạo một file Copybook `EMPLOYEE.cpy`

```
       05 USER-ID           PIC 9(05).
       05 USER-NAME         PIC X(20).
       05 USER-ROLE         PIC X(10).
       05 USER-BALANCE      PIC 9(07)V99.
```

Sử dụng lại trong chương trình khác

```
       IDENTIFICATION DIVISION.
       PROGRAM-ID. LOG-USER-DATA.

       DATA DIVISION.
       WORKING-STORAGE SECTION.

       *> Nhúng cấu trúc từ Copybook vào một Group Item
       01  WS-USER-RECORD.
           COPY "USERDATA.cpy".

       PROCEDURE DIVISION.
       MAIN-PROCEDURE.
           *> 1. Gán giá trị mẫu cho các biến trong Copybook
           MOVE 12345          TO USER-ID
           MOVE "GEMINI AI USER" TO USER-NAME
           MOVE "ADMIN"        TO USER-ROLE
           MOVE 1500.50        TO USER-BALANCE

           *> 2. Log dữ liệu ra màn hình (Console Log)
           DISPLAY "--- USER TRANSACTION LOG ---"
           DISPLAY "ID     : " USER-ID
           DISPLAY "NAME   : " USER-NAME
           DISPLAY "ROLE   : " USER-ROLE
           DISPLAY "BALANCE: " USER-BALANCE
           DISPLAY "----------------------------"

           STOP RUN.
```
