## Toán tử trong COBOL

`[]`: tham số tuỳ chọn, không bắt buộc  
`{}`: tham số bắt buộc

### ADD - Phép cộng

---

```cobol
ADD [CORRESPONDING/CORR] {Identifier1/literal1} TO {identifier2} GIVING {Identifier3} [ROUNDED]
[ON SIZE ERROR / NOT ON SIZE ERROR imperative-statement]
END-ADD
```

ADD dùng để thực hiện phép cộng.

Trong đó:

- `CORRESPONDING / CORR`: Trong toán tử, chỉ dùng cho ADD và SUBTRACT, cộng các trường có cùng tên trong hai nhóm dữ liệu khác nhau.
  ```cobol
  01 GRP-A.
      05 NUMBER1   PIC 9(3) VALUE 300.
  01 GRP-B.
      05 NUMBER1   PIC 9(3) VALUE 200.
  ADD CORRESPONDING GRP-A TO GRP-B.       # NUMBER1 OF B -> 500
  ```
  - Biến không tồn tại ở một trong 2 group sẽ bị bỏ qua, tức giá trị không thay đổi
- `GIVING`: Lưu kết quả tính vào biến phía sau.
- `ROUNDED`: Làm tròn kết quả nếu phần thập phân vượt quá định nghĩa của biến lưu trữ.
  - Cắt bỏ tất cả các chữ số thừa bên phải
  - Làm tròn số cuối cùng lên nếu chữ số tiếp theo >= 5
  - Nếu chữ số tiếp theo < 5 thì giữ nguyên.
- `ON SIZE ERROR`: Xử lý khi kết quả bị tràn dữ liệu.
- `NOT ON SIZE ERROR`: Xử lý khi kết quả không bị tràn dữ liệu.
- `END-ADD`: Từ khóa kết thúc lệnh ADD (thường dùng trong các cấu trúc lệnh lồng nhau).

  ```cobol
  01 GP-1.
    02 A    PIC 9(2) VALUE 50.
    02 B    PIC 9(2) VALUE 60.
    02 C    PIC 9(2).
    02 D    PIC 9(2).
  ...
  ADD A TO B.                   # B = A + B = 10 (110 nhưng chỉ nhận 2 số sau)
  ADD A TO B ROUNDED.           # B = A + B = 60 (B lúc này là 10)
  ADD A TO B GIVING C D.        # C = d = A + B = 10 (110 nhưng chỉ nhận 2 số sau), A và B không đổi
  ADD A TO B ROUNDED ON SIZE ERROR DISPLAY "ON SIZE ERROR". # ON SIZE ERROR vì A + B - 110 vượt quá 9(2)
  ```

### SUBTRACT - Phép trừ

---

```cobol
SUBTRACT [CORRESPONDING/CORR] {Identifier1/literal1} FROM {identifier2} GIVING {Identifier3} [ROUNDED]
[ON SIZE ERROR / NOT ON SIZE ERROR imperative-statement]
END-SUBTRACT
```

SUBTRACT dùng để thực hiện phép trừ.

```cobol
01 GP-1.
    02 A    PIC 9(2) VALUE 50.
    02 B    PIC 9(2) VALUE +60.
    02 C    PIC 9(2).
    02 D    PIC 9(2).
...

SUBTRACT A FROM B.                      # B = B - A = 1{
SUBTRACT A FROM B ROUNDED.              # B = 4}  # Lúc này B = 1{
SUBTRACT A FROM B ROUNDED ON SIZE ERROR DISPLAY "ON SIZE ERROR". # DISPLAY NOTHING
SUBTRACT A FROM B GIVING C D.           # C = D = B - A = 40
```

### MULTIPLY - Phép nhân

---

```cobol
MULTIPLY {Identifier1/literal1} BY {identifier2} GIVING {Identifier3} [ROUNDED]
[ON SIZE ERROR / NOT ON SIZE ERROR imperative-statement]
```

MULTIPLY dùng để thực hiện phép nhân.

```cobol
01 GP-1.
    02 A    PIC 9(2) VALUE 50.
    02 B    PIC 9(2) VALUE 60.
    02 C    PIC 9(2).
    02 D    PIC 9(4).
...

MULTIPLY A BY B.                      # B = B * A = 00 (3000 nhưng chỉ nhận 2 số sau)
MOVE 60 TO B.                         # B = 60
MULTIPLY A BY B ROUNDED NOT ON SIZE ERROR DISPLAY "NOT ON SIZE ERROR" MOVE 10 TO A. # DISPLAY NOTHING, A = 50, B = 60
MULTIPLY A BY B ROUNDED ON SIZE ERROR DISPLAY "ON SIZE ERROR" MOVE 10 TO A.         # ON SIZE ERROR, A = 10, B = 60
MULTIPLY A BY B GIVING C D.           # C = 00 (600 nhưng chỉ nhận 2 số sau), D = 0600
```

### DIVIDE - Phép chia

```cobol
DIVIDE {Identifier1/literal1} {INTO/BY} {identifier2} GIVING {Identifier3} [ROUNDED] {REMAINDER Identifier4}
[ON SIZE ERROR / NOT ON SIZE ERROR imperative-statement]
```

DIVIDE dùng để thực hiện phép chia.

Lưu ý:

- Sử dụng `BY` phải đi kèm với `GIVING`, nếu không sẽ gây lỗi biên dịch
- Trong trường hợp chia cho 0, chương trình sẽ báo lỗi biên dịch `ABENDED ...`
- `DIVIDE A INTO B.` -> B = B / A
- `DIVIDE A BY B GIVING C.` -> C = A / B
- REMAINDER = Số bị chia - (Thương số x Số chia)

```cobol
01 GP-1.
    02 A    PIC 9(2) VALUE 50.
    02 AA   PIC 9(2) VALUE 01.
    02 B    PIC 9(2) VALUE 60.
    02 BB   PIC 9(2) VALUE 02.
    02 C    PIC 9(2).
    02 CC   PIC 9(2).
    02 D    PIC 9(4).
    02 DD   PIC 9(4).
...

DIVIDE A INTO B.                      # B = B / A = 01 (60 / 50 = 1.2)
MOVE 60 TO B.                         # B = 60
DIVIDE A INTO B GIVING AA REMAINDER BB
    NOT ON SIZE ERROR DISPLAY "NOT ON SIZE ERROR"
    MOVE 00 TO A.                     # NOT ON SIZE ERROR, A = 00, B = 60, AA = 01, BB = 10
DIVIDE A INTO B ROUNDED
    ON SIZE ERROR DISPLAY "ON SIZE ERROR"
    MOVE 10 TO A.                     # ON SIZE ERROR, A = 10, B = 60

MOVE 40 TO A.
MOVE 60 TO B.
DIVIDE A INTO B GIVING C D.           # C = 01, D = 0001
DIVIDE A BY B GIVING CC REMAINDER DD. # CC = 00 (40 / 60 = 0.66667), DD = 0040 (40 - (0*60) = 40)
```

### COMPUTE - Phép tính tổng quát

---

```cobol
COMPUTE {Identifier1/literal1} [ROUNDED] = {ALL arithmetic operations}
[ON SIZE ERROR / NOT ON SIZE ERROR imperative-statement]
```

COMPUTE dùng để thực hiện biểu thức tính toán phức tạp ở vế bên phải dấu bằng và gán giá trị cho biến bên trái dấu bằng.

```cobol
01 GP-1.
    02 A    PIC 9(2).
    02 B    PIC 9(2) VALUE 15.
    02 C    PIC 9(2) VALUE 10.
    02 D    PIC 9(2).
    02 E    PIC 9(2).
    02 F    PIC 9(2).
...
COMPUTE A = B - C.                  # A = 15 - 10 = 05
COMPUTE D ROUNDED = A + B.          # D = 20
COMPUTE E = (A * B)/(D - C) NOT ON SIZE ERROR DISPLAY "NOT ON SIZE ERROR".  # NOT ON SIZE ERROR
COMPUTE F = B * C ON SIZE ERROR DISPLAY "ON SIZE ERROR".                    # ON SIZE ERROR
```
