## File trong COBOL

File được sử dụng để lưu trữ dữ liệu trong COBOL dưới dạng DISK (đĩa) hoặc TAPE (băng từ).
File được chia thành các record, mỗi record được chia thành các field, chứa thông tin về dữ liệu.

COBOL hỗ trợ 3 loại file:

- File tuần tự (Sequential File)
- File chỉ mục (Indexed File)
- File tổ chức tương đối (Relative Organization File)

Trong COBOL, hỗ trợ 3 chế độ truy cập (ACCESS MODE):

- SEQUENTIAL: Đọc tuần tự từ đầu đến cuối (theo thứ tự bảng chữ cái/số của Key).
- RANDOM: Truy cập một record một cách ngẫu nhiên, có thể nhảy thẳng đến một record cụ thể bằng Key.
- DYNAMIC: Là sự kết hợp của Sequential access và Random access. Vừa có thể đi tuần tự, vừa nhảy cóc đến một record bất kỳ.

### 1. Sequential File (File tuần tự)

Sequential File còn được gọi là Flat file. Trong loại file này, các record được lưu theo thứ tự tuần tự, record mới luôn được thêm vào cuối file theo.

Để truy cập record thứ N, chúng ta phải đọc trước (N-1) record nên thường sử dụng loại file này khi chỉ cần đọc và ghi đơn giản, và việc tìm kiếm record không diễn ra thường xuyên.

Sequential File có 2 dạng record length là:

- Fixed-length record: Mỗi record trong file có cùng một số byte. Nếu dữ liệu thực tế ngắn hơn, phần còn lại sẽ được padding bằng spaces.
  - Ưu điểm: Dễ tính vị trí record. Xử lý nhanh và đơn giản nên thường dùng để xử lý batch.
  - Nhược điểm: Tốn dung lượng nếu dữ liệu không đủ độ dài.
- Variable-length record: Mỗi record có thể có độ dài khác nhau, chỉ chiếm đúng số byte cần thiết. Thường có thêm thông tin độ dài record ở đầu record.
  - Ưu điểm: Tiết kiệm dung lượng, linh hoạt hơn khi dữ liệu không đồng đều.
  - Nhược điểm: Không thể tính vị trí record bằng công thức, phải đọc tuần tự.

#### 1.1. Line Sequential File

Line Sequential là một dạng đặc biệt của Sequential file, trong đó mỗi record được phân tách bằng ký tự xuống dòng `Carriage Return (X“0D”)` hoặc `Line Feed (X“0A”)` ở cuối ký tự không phải khoảng trắng cuối cùng.

File Line Sequential luôn chứa các record có độ dài biến đổi và còn được gọi là file văn bản (Text file). Đối với file báo cáo (Report file), chúng ta nên định nghĩa file dưới dạng Line Sequential vì nó đảm bảo các yếu tố như dữ liệu hiển thị theo từng dòng, độ dài thay đổi, có ký tự xuống dòng...

```
CUSTOMER REPORT
------------------------
ID    NAME        BALANCE
001   TANAKA      10000
002   SATO        20000
```

#### Syntax

```cobol
ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
SELECT LOGICAL_FL ASSIGNED TO PHYSICAL_FL
    ORGANIZATION IS SEQUENTIAL. *> Hoặc LINE SEQUENTIAL
DATA DIVISION.
FILE SECTION.
FD LOGICAL_FL.
01 LOGICAL-REC.
    05 STORE-ID     PIC 9(05).
    05 FILLER       PIC X(01).
    05 ITEM-ID      PIC X(10).
    05 FILLER       PIC X(64).
```

Phần khai báo này phải được viết trong FILE-CONTROL SECTION, nằm dưới INPUT-OUTPUT SECTION, và INPUT-OUTPUT SECTION được định nghĩa trong ENVIRONMENT DIVISION.

**1.2. Mệnh đề FD (File Description)**

```cobol
DATA DIVISION.
FILE SECTION.
FD FILE-NAME.   *> Tên file ở đây phải trùng với tên đã SELECT trong FILE-CONTROL.
[RECORD CONTAINS N characters]
[BLOCK CONTAINS I RECORDS]
[DATA RECORD IS RECORD-DET]
[RECORDING MODE IS {F/V/U}].
```

- `RECORD CONTAINS N characters` - Mô tả kích thước của Record

  ```cobol
  *> Format 1 - dùng cho Fixed-Length files
  RECORD CONTAINS 80 CHARACTERS

  *> Format2 - dùng cho Variable-Length files
  RECORD CONTAINS SIZE N-1 TO N-2 CHARACTERS

  *> Format3 - dùng cho Dynamic Files
  RECORD IS VARYING IN SIZE N-1 TO N-2 DEPENDING ON DATA-ITEM-NAME
  ```

- `BLOCK CONTAINS I RECORDS` - Xác định kích thước Block của file, xác định kích thước của physical record. Chỉ định số record trong một block ghi ra đĩa trong mỗi lần I/O.

  ```cobol
  *> Format 1 - dùng cho Fixed-Length files
  BLOCK CONTAINS 80 CHARACTERS

  *> Format2 - dùng cho Variable-Length files
  BLOCK CONTAINS SIZE N-1 TO N-2 CHARACTERS

  *> Format3 - dùng cho Dynamic Files
  BLOCK IS VARYING IN SIZE N-1 TO N-2 DEPENDING ON DATA-ITEM-NAME

  *> Special - Dành cho sequentail (QSAM) files. Block size sẽ được hệ thống xác định tại thời điểm thực thi
  BLOCK CONTAINS 0 CHARACTERS
  ```

- `DATA RECORD IS RECORD-DET` - Liên kết file với cấu trúc record trong WORKING-STORAGE, giống kiểu Schema cho record

- `RECORDING MODE IS {F/V/U}` - Chỉ định cách lưu record.

  F-Fixed Length, V-Variable Length, U-Undefined (Hệ thống tự quyết định, ít dùng)

### 2. Indexed File (File chỉ mục)

Indexed file là loại file có thể truy cập nhanh hơn nhờ việc truy cập được thực hiện thông qua giá trị khóa (key), thứ được tạo từ ký tự chữ và số (alphanumeric key) và là định danh duy nhất.

Lệnh `ORGANIZATION IS INDEXED` dùng để chỉ định File sẽ được tổ chức theo cấu trúc có chỉ mục (index structure). Dữ liệu không chỉ lưu tuần tự mà hệ thống sẽ tạo một cấu trúc index nội bộ (thường giống B-Tree) để:

- Tìm record theo KEY nhanh
- Truy cập ngẫu nhiên
- Duy trì thứ tự key

Lưu ý:

- Nếu cần ghi vào Indexed file, chúng ta phải ghi record theo thứ tự tăng dần của key.  
  Ví dụ: Để ghi record có key A123, chúng ta phải ghi tất cả record có key nhỏ hơn A123 trước.
- Key phân biệt chữ hoa và chữ thường, vì vậy A123 khác a123.
- Không thể cập nhật (thay đổi) trường key trong Indexed file.
- Có thể xóa một record cụ thể. Record đó không bị xóa vật lý khỏi bộ nhớ, nhưng index sẽ cập nhật để không còn trỏ tới record đó nhằm tránh phải dịch chuyển dữ liệu phía sau gây tốn chi phí I/O (giống soft-delete trong SQL)
  - Muốn giải phóng / xóa record thực sự phải rebuild file hoặc Reorganize file.

#### Syntax

```cobol
SELECT LOGICAL-FILE ASSIGNED TO PHYSICAL-FILE
ORGANIZATION IS INDEXED
ACCESS MODE IS SEQUENTIAL *> RANDOM / DYNAMIC
RECORD KEY IS I-KEY
ALTERNATE KEY IS IA-KEY.
```

- `RECORD KEY IS ...`: Định nghĩa primary key, duy nhất, hệ thống sẽ tạo index dựa trên khóa này.
- `ALTERNATE KEY IS ...`: Định nghĩa khóa phụ, có thể trùng, cho phép truy cập các bản ghi bằng alternate key thay vì primary key.

### 3. Relative Organization File (File tổ chức tương đối)

Còn được gọi là file Relative Record Data Set (RRDS). Các record trong Relative File có thể được truy cập bằng cách sử dụng RRN (Relative Record Number – Số thứ tự bản ghi tương đối, tức số thứ tự vật lý của record trong file).

Giả sử chúng ta có yêu cầu đọc một record với một giá trị key cụ thể mà chúng ta đã biết trước, thì nên định nghĩa file dưới dạng RRDS.

Chúng ta có thể truy cập record theo bất kỳ thứ tự nào bằng cách khai báo RELATIVE KEY. Ở đây RELATIVE KEY chỉ được phép là kiểu số (numeric key) và nó là cầu nối để trỏ đến RRN.

Vd: Khi muốn đọc bản ghi số 10 (RRN = 10), cần gán giá trị 10 vào biến RELATIVE KEY rồi thực hiện lệnh READ.

**Vì sao không đọc trực tiếp RRN mà phải thông qua trung gian là RELATIVE KEY?**

Quy tắc vàng: "Đừng bao giờ để phần mềm can thiệp trực tiếp vào địa chỉ vật lý của phần cứng".  
RRN là một khái niệm của hệ thống tệp (File System). Nó đại diện cho một vị trí cụ thể trên ổ đĩa nên phải dùng RELATIVE KEY để tạo một lớp Mask để nói với hệ thống rằng "Tôi muốn bản ghi số 10", còn việc bản ghi số 10 đó nằm ở byte thứ bao nhiêu, track nào trên ổ cứng thì đó là việc của Hệ thống quản lý tệp.

#### Syntax

```cobol
SELECT LOGICAL-FILE ASSIGNED TO PHYSICAL-FILE
ORGANIZATION IS RELATIVE
ACCESS MODE IS SEQUENTIAL *> RANDOM / DYNAMIC
RECORD KEY IS R-KEY.
```

- `RECORD KEY IS ...`: Định nghĩa Relative Record Number.

Giả sử chúng ta có 9000 bản ghi trong một file.  
Để chứa 1000 bản ghi, chúng ta phải định nghĩa key với PIC 9(4), có thể chứa giá trị từ 0001 đến 9999.

```cobol
SELECT LOGICAL-FILE ASSIGNED TO PHYSICAL-FILE
ORGANIZATION IS RELATIVE
ACCESS MODE IS RANDOM
RECORD KEY IS R-KEY.
...
FD LOGICAL-FL.
01 LOGICAL-FL-REC.
   05 R-KEY PIC 9(4).
   05 R-NAME PIC X(20).
```

## Phương pháp truy cập File

QSAM (Queued Sequential Access Method) – phương pháp truy cập file tuần tự trên hệ thống mainframe IBM dùng trong batch processing. Nó giúp quản lý Buffer, Block I/O, đọc/ghi vật lý ra đĩa. Là file tuần tự, không có index, đọc từ đầu đến cuối.

VSAM (Virtual Storage Access Method) - phương pháp truy cập và quản lý file do IBM phát triển cho hệ thống mainframe (z/OS) được dùng để lưu trữ và truy xuất dữ liệu hiệu năng cao. Có thể truy cập trực tiếp theo key, có index, có xử lý INVALID KEY.

Ngoài ra còn có 3 loại ít dùng:

- BSAM (Basic Sequential Access Method): Mức thấp hơn QSAM, dev phải tự quản lý buffer
- BDAM (Basic Direct Access Method): Truy cập trực tiếp theo block, không có index
- ISAM (Indexed Sequential Access Method): Tiền thân của VSAM, có index nhưng cấu trúc cố định.

## Thao tác với File

Một file phải được mở (OPEN) trong PROCEDURE DIVISION trước khi thực hiện bất cứ thao tác nào. Sau khi mở thành công, chương trình có thể thực hiện các chức năng READ / WRITE / REWRITE / DELETE.

Cuối cùng, file phải được đóng (CLOSE). Sau khi chương trình thực thi thành công, tất cả các file đã sử dụng sẽ tự động được đóng, đây là một best practice khi chủ động đóng file để tránh mọi sự cố.

Cú pháp:

```cobol
*> Mở file để đọc
OPEN INPUT FILE-NAME.

*> Mở file để ghi
OPEN OUTPUT FILE-NAME.

*> Mở file để đọc, ghi và ghi đè (rewrite)
OPEN I-O FILE-NAME.

*> Mở file để ghi nối thêm (append). Dùng cho file Relative và Indexed.
OPEN EXTEND FILE-NAME.
```

### 1. READ

```cobol
READ FILE-NAME [NEXT / PREVIOUS] RECORD [INTO IDENTIFIER-1]
    [AT END {imperative statement}]
    [NOT AT END {imperative statement}]
    [END-READ]
```

NEXT – Bắt buộc khi dùng chế độ truy cập DYNAMIC

```cobol
READ FILE-NAME RECORD [INTO IDENTIFIER-1]
    [KEY IS KEY-1]
    [INVALID KEY {imperative statement}]
    [NOT INVALID KEY {imperative statement}]
    [AT END {imperative statement}]
    [NOT AT END {imperative statement}]
    [END-READ]
```

- `READ ... RECORD [INTO ...]`: Đọc 1 record từ file vào record area (FILE SECTION), nếu có INTO thì dữ liệu sẽ được copy sang biến chỉ định sau nó.
- `KEY IS`: Tìm key trong index, nếu tìm thấy thì trả về record tương ứng.
  - KEY, INVALID KEY và NOT INVALID KEY chỉ được sử dụng cho file indexed và relative, là các file thuộc loại VSAM (không áp dụng cho file sequential).
- `INVALID KEY ...`: Thực thi khi không tìm thấy key hoặc duplicate key (trong WRITE/REWRITE) hoặc relative record không tồn tại.
- `NOT INVALID KEY ...`: Thực thi khi đọc thành công theo key.
- `AT END ...`: Thực thi khi đọc tuần tự đến hết file.
- `NOT AT END ...`: Thực thi khi đọc tuần tự nhưng chưa đến cuối file.
- `END-READ`: Kết thúc block READ, tránh sự mơ hồ khi có nested IF

### 2. START

Sử dụng khi muốn đặt con trỏ (pointer) tại một vị trí cụ thể trong file có tổ chức INDEXED hoặc RELATIVE.

START không lấy dữ liệu của record, nó chỉ đặt con trỏ đến record phù hợp để chuẩn bị đọc. Chỉ sử dụng được nếu file được mở ở chế độ I-O hoặc INPUT và Access mode là Sequential hoặc Dynamic.

```cobol
START FILE-NAME
    [KEY IS {= / EQUAL TO / GREATER THAN /> /
             LESS THAN / < / NOT LESS THAN / NOT < THAN ...} KEY-1]
    [INVALID KEY {imperative statement}]
    [NOT INVALID KEY {imperative statement}]
    [END-START]
```

### 3. WRITE

```cobol
WRITE FILE-RECORD
    [FROM IDENTIFIER-1 / literal-1]
    [BEFORE / AFTER ADVANCING {identifier-2 / number-2 / PAGE} LINE / LINES]
    [KEY IS KEY-1]
    [INVALID KEY {imperative statement}]
    [NOT INVALID KEY {imperative statement}]
    [AT END-OF-PAGE / EOP {imperative statement}]
    [NOT AT END-OF-PAGE / EOP {imperative statement}]
    [END-WRITE]
```

- `BEFORE ADVANCING`: Dùng cho Report khi dòng phải được in trước khi sang trang mới. Dùng cho in header đầu trang, khi không muốn ghi đè dòng hiện tại.
- `AFTER ADVANCING`: Dùng cho Report khi dòng phải được in sau khi đã sang trang mới. Ghi đè dòng hiện tại, sau đó xuống dòng, tương tự như println().

Lưu ý: FILE-NAME và FILE-RECORD là khác nhau, FILE-RECORD là record description khai báo trong FILE SECTION, đại diện vùng bộ nhớ 1 record của file buffer.

### 4. REWRITE

Dùng để ghi đè lại record vừa được đọc gần nhất đang mở ở chế độ I-O, không thể REWRITE nếu chưa READ trước đó.

```cobol
REWRITE FILE-RECORD
    [FROM identifier-1 / literal-1]
    [INVALID KEY imperative-statement]
    [NOT INVALID KEY imperative-statement]
END-REWRITE
```

### 5. DELETE

Dùng để xóa một record trong file đang mở ở chế độ I-O.

```cobol
DELETE file-name RECORD
    [INVALID KEY imperative-statement]
    [NOT INVALID KEY imperative-statement]
END-DELETE
```

### 6. CLOSE

Dùng để đóng file một cách tường minh.

```cobol
CLOSE file-name
```
