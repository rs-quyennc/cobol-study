## Internal presentation of data

### 1. DISPLAY

Cú pháp: `USAGE IS DISPLAY`

Mặc định, USAGE là DISPLAY. Các bản ghi được lưu dưới định dạng ASCII, mỗi chữ số chiếm 1 byte.

Ví dụ:

```
01 A PIC S9(2)V9(2) USAGE IS DISPLAY.
```

Biến này chiếm 4 bytes trong bộ nhớ.

- S : dấu + - (không chiếm Byte)
- 9(2) : 2 số nguyên
- V : dấu thập phân (không chiếm Byte)
- 9(2) : 2 số sau thập phân

```
01 B PIC 9(6) USAGE IS DISPLAY.
```

Biến này chiếm 6 bytes trong bộ nhớ.

| Type         | Ký hiệu | Số byte |
| ------------ | ------- | ------- |
| Numeric      | 9       | 1       |
| Alphabetic   | A       | 1       |
| Alphanumeric | X       | 1       |

### 2. COMP (COMPUTATIONAL)

COMP được lưu dưới dạng nhị phân (Binary format). COMP chỉ dùng cho số nguyên (không dùng cho số thập phân), không lưu từng chữ số nên sẽ tiết kiệm bộ nhớ hơn.

Được chia thành 4 loại:

| Type          | Ý nghĩa tương đương                           | Byte         |
| ------------- | --------------------------------------------- | ------------ |
| COMP / COMP-4 | Số dạng nhị phân                              | 2 / 4 / 8    |
| COMP-1        | Hexadecimal (single precision floating point) | 4            |
| COMP-2        | Hexadecimal (double precision floating point) | 8            |
| COMP-3        | Packed decimal (thập phân nén)                | (digits+1)/2 |

Ví dụ:

```
01 A PIC 9(6) USAGE IS COMP.
```

Quy tắc:

- Nếu Z = 1 đến 4 → chiếm 2 bytes
- Nếu Z = 5 đến 9 → chiếm 4 bytes
- Nếu Z = 10 đến 18 → chiếm 8 bytes

→ Vì 6 nằm trong khoảng 5–9 → chiếm 4 bytes

```
01 A USAGE COMP-1.
```

→ Mặc định 4 bytes, không chứa mệnh đề PICTURE.

```
01 A USAGE COMP-2.
```

→ Mặc định 8 bytes, không chứa mệnh đề PICTURE.

```
PIC 9(5) COMP-3
```

→ bytes = (5/2) + 1/2 = 3 bytes

Quy tắc:

- Mỗi digit (chữ số) chiếm ½ byte.
- 1 byte = 2 nibble, nên ½ byte = 1 nibble.
- Nếu có dấu (+ hoặc -) thì dấu sẽ được lưu ở nibble ngoài cùng bên phải.

Ngay cả khi số có dấu hoặc không dấu, COMP-3 luôn dành một nibble cho dấu, kể cả khi giá trị được khai báo là unsigned.

Cách tính số byte:

- Nếu Z là số chẵn: `bytes = (Z / 2) + 1`
- Nếu Z là số lẻ: `bytes = (Z / 2) + 1/2`

**Cách COBOL lưu số nguyên khi dùng USAGE COMP**

| Picture         | Biểu diễn lưu trữ           | Giá trị số                                              |
| --------------- | --------------------------- | ------------------------------------------------------- |
| S9(1) – S9(4)   | Binary halfword (2 bytes)   | -32768 → +32767                                         |
| 9(1) – 9(4)     | Binary halfword (2 bytes)   | 0 → 65535                                               |
| S9(5) – S9(9)   | Binary fullword (4 bytes)   | -2,147,483,648 → +2,147,483,647                         |
| 9(5) – 9(9)     | Binary fullword (4 bytes)   | 0 → 4,294,967,295                                       |
| S9(10) – S9(18) | Binary doubleword (8 bytes) | -9,223,372,036,854,775,808 → +9,223,372,036,854,775,807 |
| 9(10) – 9(18)   | Binary doubleword (8 bytes) | 0 → 18,446,744,073,709,551,615                          |
